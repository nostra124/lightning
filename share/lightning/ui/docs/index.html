⚡ Lightning Wallet — Help & API docs

Covers the PWA, the REST API, and the MCP surface. CLI and node operation are documented in the man pages (man lightning) and share/doc/lightning/.

Using the wallet

  1. Open the wallet URL and tap Create my first account. The node returns an account_id and a one-time api_key.
  2. Save the API key — it is shown once and is your backup credential.
  3. Top up shows your on-chain address; the node credits deposits within ~1 minute.
  4. Send pays a pasted BOLT-11; Receive mints one to share.

Credentials: the lt_… API key is the bearer (also used by LLM agents / CLI). Passkey login is a follow-up backend.

REST API

Base path /.well-known/lightning/v1. Authenticated endpoints take Authorization: Bearer <api_key>. Business-rule failures return 402, auth 401, bad input 400.

POST   /accounts                      create (anonymous)
GET    /accounts/<id>/balance         { balance_sat, limit_sat, overdraft }
GET    /accounts/<id>/topup[?sat=N]   { address, uri }  (BIP-21)
POST   /accounts/<id>/withdraw        { sat, address }
POST   /accounts/<id>/pay             { target, sat? }
POST   /accounts/<id>/recv            { sat, description }  -> { bolt11 }
POST   /accounts/<id>/recv-reusable   BOLT-12 offer (sat may be "any")
POST   /accounts/<id>/transfer        { to, sat, note }   intra-node
GET    /accounts/<id>/referrals
POST   /accounts/<id>/close
POST   /accounts/<id>/invoice         commercial invoice + terms (FEAT-225)
.../standing-orders                  recurring push payment   (FEAT-226)
.../mandates                         direct debit             (FEAT-227)
.../charges                          charge lifecycle         (FEAT-228)
GET    /accounts/<id>/export/tax-data?year=YYYY&format=csv|json (FEAT-230)
GET    /price?base=EUR               public sat/fiat tick     (FEAT-229)

Example:

curl -X POST https://HOST/.well-known/lightning/v1/accounts -d '{"hint":"pocket"}'
curl https://HOST/.well-known/lightning/v1/accounts/<id>/balance \
  -H "Authorization: Bearer lt_…"

MCP (for LLM agents)

Endpoint POST /.well-known/lightning/v1/mcp; descriptor GET /.well-known/lightning/mcp.json; bearer-authed with an account API key. JSON-RPC 2.0:

{ "jsonrpc":"2.0","id":1,"method":"tools/call",
  "params":{"name":"balance","arguments":{"account_id":"bc1…"}} }

tools/list enumerates the tools (balance, topup, pay, recv, transfer, …), mirroring the REST endpoints. To brief an agent, hand it the base URL, an API key, and llms.txt.
